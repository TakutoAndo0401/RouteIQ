## 境界の考え方

コード上には 17 個の React component 関数があります。このうち上記 9 個を独立した公開 UI surface として扱います。`ListBlock`、`AnalysisFailureState`、履歴パネル内部などの非公開 subcomponent は、Storybook のためだけに export せず、親 component の状態 story で確認します。

`App` はテーマ永続化と `RouteComparisonPage` の起動を担う composition root です。表示内容は Page stories と重複するため、個別カタログには含めていません。

## 外部依存の扱い

<ul>
  <li><code>/api/client-config</code>、<code>/api/fuel-prices</code>、<code>/api/route-analysis</code> は MSW の固定 fixture で再現します。</li>
  <li>Google Maps は実 API key を使わず、利用不可状態または Storybook 専用の決定論的 mock で確認します。</li>
  <li>global CSS は本番と同じ順序で読み込みます。</li>
  <li>地図 modal のように <code>document.body</code> へ portal する UI もテーマが揃うよう、dark class は <code>html</code> と <code>body</code> の両方へ適用します。</li>
</ul>

## まだ component 化していないパターン

`.primary-button`、`.input-shell`、role pill、panel/card surface は複数箇所で使われていますが、現時点では独立した React component ではありません。再利用要件が増えた時点で抽出し、Shared または Widgets に追加するのが安全です。
